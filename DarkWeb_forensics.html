🌐 Tor Browser - Accessing Hidden Services

What is Tor?

Tor stands for "The Onion Router." It's a free software that enables anonymous communication by directing internet traffic through a series of volunteer-operated servers called relays. Each layer of encryption is like a layer of an onion - hence the name.

How Does Tor Work?

Layer 1 (Outer): Your computer encrypts the message with the exit relay's key
Layer 2 (Middle): Then encrypts with the middle relay's key
Layer 3 (Inner): Finally encrypts with the entry relay's key

The message travels through these three relays in reverse order. Each relay only knows the previous and next relay, never the complete path or your real IP address.

Installation Steps

1

Update Your System

First, update the package repository to ensure you have the latest software available.

sudo apt update
2

Install Tor Browser Launcher

This utility automatically downloads and manages Tor Browser for you.

sudo apt install torbrowser-launcher
3

Launch Tor Browser

Start the Tor Browser application. It will automatically connect to the Tor network.

torbrowser-launcher
4

Wait for Connection

The browser will establish connections through multiple relays. This usually takes 10-30 seconds.

Understanding .onion Addresses

.onion addresses are special domain names that only work within the Tor network. They appear as 56-character alphanumeric strings (v3 addresses) derived from cryptography.

Example: xmh57jrknzkhv6y3ls3ubitzfqnkrwxhopf5aygthi7d6rplyvk3noyd.onion

Important: Standard browsers like Chrome or Firefox cannot access .onion addresses. Only Tor Browser can.

Key Advantages of Tor

🔒 Encrypted

Multiple layers of encryption protect your data

👁️ Anonymous

Destination servers don't see your real IP

🛡️ Decentralized

No central point of failure or control

✅ Free

Open-source and maintained by volunteers

🔗 ProxyChains - Advanced Proxy Routing

What is ProxyChains?

ProxyChains is a Linux tool that forces any command-line application to route its traffic through proxy servers. This allows investigators to use tools like nmap, curl, and other utilities through anonymity networks without modifying the tools themselves.

Why Use ProxyChains?

  • Transparent Routing: Works with any command-line tool
  • Investigator Safety: Hides the investigator's real IP address
  • Multiple Layers: Chains multiple proxies together
  • Easy Configuration: Simple configuration file setup
  • Flexibility: Works with Tor, SOCKS proxies, and other services

How ProxyChains Works

Traffic Flow Through ProxyChains

Investigator
→
Proxy 1
→
Proxy 2
→
Proxy 3
→
Target

Each proxy server only knows about the previous and next proxy in the chain. No single proxy knows the complete path from the investigator to the target.

Installation and Configuration

1

Verify Tor is Running

ProxyChains typically uses Tor's SOCKS proxy, so Tor must be installed and running.

sudo service tor status

If Tor is not running, start it:

sudo service tor start
2

Install ProxyChains

Install the ProxyChains package on your system.

sudo apt install proxychains4
3

Edit Configuration File

Open the ProxyChains configuration file in a text editor.

sudo nano /etc/proxychains4.conf
4

Change Proxy Mode

Find the line with strict_chain and replace it with:

dynamic_chain

This allows ProxyChains to continue working even if one proxy fails.

5

Verify Tor SOCKS Proxy

Make sure this line exists in the configuration:

socks5 127.0.0.1 9050

Save the file (Ctrl+X, then Y in nano).

Using ProxyChains

Example 1: Web Browser

Route Firefox through ProxyChains:

proxychains firefox https://example.com

Example 2: Network Scanning

Scan a target server through ProxyChains:

proxychains nmap -p 80,443 target.com

Example 3: Check Your IP

Verify you're hidden behind a proxy:

proxychains curl https://whatismyipaddress.com

The IP address shown will be the exit proxy, not your real IP.

Proxy Mode Comparison

Mode Behavior Best For
strict_chain All proxies must be working Maximum anonymity requirement
dynamic_chain Works if proxies fail Reliability is important
random_chain Random proxy ordering Unpredictable routing patterns

🌍 Internet Layers - Surface, Deep, and Dark Web

The Three Layers of the Internet

The internet is often described in three layers based on accessibility and content type. Understanding these distinctions is crucial for digital forensics investigators.

1. Surface Web (95%)

What It Is:

The portion of the internet indexed by search engines and accessible to anyone with a standard web browser.

Examples:

  • Google, Wikipedia, Facebook
  • News websites (CNN, BBC, Reuters)
  • Shopping sites (Amazon, eBay)
  • Social media platforms
  • Email services (Gmail, Outlook)

Access:

✓ Standard browser (Chrome, Firefox, Edge)

2. Deep Web (4%)

What It Is:

Content that exists on the internet but is not indexed by search engines. It's not hidden by default - it just requires authentication or specific access credentials.

Examples:

  • Medical records (hospital systems)
  • Bank accounts and financial records
  • Academic databases and research papers
  • Legal documents and court records
  • Private email inboxes
  • Subscription-based services
  • Company databases and intranets

Access:

✓ Standard browser + Authentication (username/password)

Important: The deep web is completely legal. It includes legitimate services that simply require authentication to protect privacy.

3. Dark Web (1%)

What It Is:

A small subset of the internet that has been intentionally hidden and requires specific software (like Tor) to access. It prioritizes anonymity and encryption.

Legitimate Uses:

  • Whistleblowing platforms (journalists' sources)
  • Privacy-focused communication tools
  • Censorship circumvention in restrictive countries
  • Anonymous forums for sensitive discussions
  • Activism and human rights work

Illegal Content (What Investigators Focus On):

  • Drug marketplaces
  • Stolen data and credentials
  • Hacking tools and exploits
  • Weapon sales
  • Child exploitation material (CSAM)
  • Counterfeit goods
  • Fraud and scam operations

Access:

✓ Specialized software: Tor, I2P, or Freenet

Common Misconceptions

❌ Myth: "Deep web and dark web are the same thing"

✓ Truth: Deep web is simply non-indexed content requiring authentication. Dark web is intentionally hidden and encrypted.

❌ Myth: "Everything on the dark web is illegal"

✓ Truth: The dark web has legitimate uses like privacy protection and circumventing censorship.

❌ Myth: "Accessing the dark web is illegal"

✓ Truth: Accessing Tor or the dark web is legal in most countries. Illegal activities ON the dark web are the problem.

🔍 Dark Web Search Engines

Just like Google indexes the surface web, specialized search engines index .onion hidden services. Here are the most important ones for forensic investigations.

1. Torch

Overview:

One of the oldest and most well-known dark web search engines. It uses crawling technology to index thousands of .onion domains.

Characteristics:

  • Established: 1996 (early Tor era)
  • Index Size: Large (thousands of .onion sites)
  • Update Frequency: Moderate
  • Privacy: High (no query logging)

Access:

xmh57jrknzkhv6y3ls3ubitzfqnkrwxhopf5aygthi7d6rplyvk3noyd.onion

⚠️ Important: Open only in Tor Browser. Standard browsers cannot access .onion sites.

Best For:

General dark web reconnaissance and finding marketplace mirrors.

2. Haystak

Overview:

A specialized Tor search engine designed specifically for indexing .onion websites. It features advanced algorithms optimized for hidden service discovery.

Characteristics:

  • Specialized Engine: Built for Tor network
  • Index Size: Comprehensive
  • Update Frequency: Fast (rapid indexing)
  • Privacy: Very High (strict no-log policy)
  • Advanced Filtering: Category-based search

Access:

haystak5njsmn2hqkewecpaxetahtwhsbsa64jom2k22z5afxhnpxfid.onion

Best For:

Identifying newly created marketplaces and rapidly changing dark web infrastructure. Preferred for active threat detection.

3. OnionLand

Overview:

A unique hybrid search engine accessible from both the regular web (clearnet) AND Tor. This allows researchers to search dark web content without directly connecting to Tor.

Characteristics:

  • Dual Access: Clearnet and Tor versions
  • Index Size: Moderate
  • Update Frequency: Slower (periodic indexing)
  • Privacy: Moderate (clearnet version)
  • Ease of Use: Very accessible

Access (Clearnet):

https://onionlandsearchengine.com

Open this in any standard browser (Chrome, Firefox, Edge).

Best For:

Quick preliminary research and reconnaissance without needing to launch Tor Browser. Perfect for initial threat intelligence gathering.

Search Engine Comparison

Search Engine Access Update Speed Best For
Torch Tor Only Moderate General reconnaissance
Haystak Tor Only Very Fast Threat detection
OnionLand Clearnet + Tor Slower Quick research

Search Strategy

Recommended Workflow:

  1. Phase 1: Use OnionLand (clearnet) for initial reconnaissance
  2. Phase 2: Launch Tor Browser and use Haystak for active threat detection
  3. Phase 3: Use Torch for secondary verification and archive searches
  4. Phase 4: Document all findings with evidence preservation

🔍 Dark Web Forensics Investigation Guide

Legal and Ethical Requirements

⚠️ Critical: All dark web investigations must be conducted with proper legal authorization. This includes:

  • Warrant or court order (where required)
  • Written authorization from organizational leadership
  • Compliance with jurisdictional laws
  • Adherence to professional ethics codes

Unauthorized investigation is illegal and violates computer crime statutes.

Investigation Workflow

1

Obtain Legal Authorization

Before accessing any dark web infrastructure, ensure you have proper legal authority through:

  • Court-issued search warrant
  • Written organizational authorization
  • Law enforcement request (if applicable)
  • Incident response authorization
2

Prepare Isolated System

Set up a dedicated forensic workstation:

  • Use a virtual machine (isolated from production network)
  • Install necessary tools (Tor, ProxyChains)
  • Disable unnecessary network services
  • Enable logging and monitoring
  • Document system specifications
3

Establish Anonymity Infrastructure

Configure protective layers:

  • Start Tor daemon and verify connection
  • Configure ProxyChains if needed
  • Test anonymity (verify no IP leakage)
  • Document all configuration details
4

Conduct Preliminary Research

Gather initial intelligence:

  • Use OnionLand for initial reconnaissance
  • Identify suspected marketplaces or forums
  • Document .onion addresses and descriptions
  • Note last-seen dates from search engine caches
5

Access Target Platforms

Connect to identified dark web sites:

  • Launch Tor Browser with fresh circuit
  • Navigate to target .onion address
  • Observe platform structure and content
  • Avoid suspicious downloads or interactions
  • Take detailed screenshots with timestamps
6

Collect Evidence

Document findings systematically:

  • Capture screenshots of relevant pages
  • Record URLs and .onion addresses
  • Document user-generated content (listings, usernames)
  • Preserve HTML source (where legally permissible)
  • Record all timestamps in UTC
7

Maintain Chain of Custody

Ensure evidence integrity:

  • Document every action taken (investigation log)
  • Record dates, times, and system states
  • Note who accessed the evidence
  • Calculate hash values (MD5, SHA-256)
  • Store evidence in secure, controlled location
8

Complete Forensic Analysis

Analyze collected information:

  • Correlate evidence across sources
  • Identify threat actors and infrastructure
  • Assess risks and impacts
  • Create detailed findings report
  • Provide actionable recommendations

Critical Documentation Requirements

Investigation Log Must Include:

  • Investigation authorization (warrant/letter)
  • Investigator identification and credentials
  • Date, time, and timezone (UTC preferred)
  • System used and configuration details
  • Tool versions and settings
  • URLs accessed and content observed
  • Screenshots with metadata
  • Chain of custody for all evidence
  • Any deviations from protocol
  • Final assessment and conclusions

Safety Best Practices

During Investigation:

  • Never download files unless absolutely necessary and with extreme caution
  • Disable plugins that could leak your real IP (JavaScript, Flash)
  • Use fresh Tor circuit for each new investigation
  • Never maximize browser window (prevents website fingerprinting)
  • Assume everything is malicious until proven safe
  • Keep detailed logs of all activities
  • Maintain isolation - don't access from production networks

Common Investigation Scenarios

Objective:

Identify and monitor dark web forums discussing your organization.

Actions:

  1. Search OnionLand for forums mentioning company name
  2. Use Haystak to identify active discussion threads
  3. Access forums via Tor Browser
  4. Monitor user discussions and leaked data mentions
  5. Document threat actors and their tactics
  6. Report findings to security team

Objective:

Investigate suspected dark web marketplace selling stolen corporate data.

Actions:

  1. Obtain warrant for investigative access
  2. Search for marketplace using Torch and Haystak
  3. Document marketplace structure and listings
  4. Capture product listings and pricing
  5. Note vendor feedback and history
  6. Correlate with known data breaches
  7. Forward evidence to law enforcement

Objective:

Investigate suspected leaked data following a security incident.

Actions:

  1. Search dark web for mentions of breach
  2. Identify data dump locations and dumps
  3. Document file listings without downloading
  4. Assess data exposure scope
  5. Identify threat actors responsible
  6. Gather evidence for legal action
  7. Coordinate with law enforcement
